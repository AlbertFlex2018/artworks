## 1.本类、接口、枚举名称
# Process 进程类[抽象]  

## 2.本类设计目的
为可以自由更新或者可以长时间存在的行为提供支持。并且进程以进程链为组成成分，可以创建链结构的进程链，以供方便以流形式运行进程。

## 3.本类的职责概要
提供进程的知识。外与进程管理器，或者自定义的代码交互，内部与只有自己了解的内容交互，更新自己的内容。以链形式组织。提供通用的可行的逻辑控制。
